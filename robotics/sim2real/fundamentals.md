# 基础问题与差距来源

Sim2Real 的核心问题是：仿真器和真实世界总是存在差距。仿真中训练出来的策略、感知模型或控制器，只有跨过这些差距后，才能在真实机器人系统中稳定工作。

![[assets/sim2real-transfer.svg|697]]

## Sim2Real Gap

Sim2Real gap 可以分成三类：

- 感知差距：仿真图像和传感器数据通常更干净，而真实世界存在噪声、曝光变化、运动模糊、遮挡、反光和传感器误差。
- 物理差距：真实系统中的摩擦、阻尼、刚度、接触、关节间隙、柔性结构和执行器延迟，很难在仿真中完整复现。
- 任务分布差距：仿真中的场景、物体、初始状态、扰动和目标分布，可能与真实任务不一致。

工程上不要只问“仿真够不够真”，更要问“策略真正依赖的变量，在真实系统中是否稳定、可观测、可控制”。

## 相机 Gap 例子

相机标定是解决感知差距的典型工程环节。仿真里相机通常拥有理想化参数：焦距、主点、畸变、外参、安装位置和坐标系关系都是确定的；真实相机则会受到镜头畸变、装配误差、传感器噪声、曝光变化和手眼坐标误差影响。

[rayheto/mono-calib](https://github.com/rayheto/mono-calib) 是一个用于缩小相机感知 gap 的实际项目。它对应的问题不是“让视觉模型更大”，而是先把真实相机的几何关系校准清楚，让真实图像、机器人坐标系和仿真坐标系尽可能对齐。

它在 Sim2Real 中的意义：

- 内参标定：校准焦距、主点和畸变参数，减少真实图像和理想针孔相机模型之间的差异。
- 外参与手眼标定：建立相机坐标系、末端执行器坐标系和机器人基坐标系之间的变换关系。
- 图像矫正：减少畸变导致的像素位置误差，让检测、跟踪和位姿估计更接近仿真假设。
- Sim vs Real 对齐：将真实相机观测和仿真视角对齐，降低策略或视觉模型迁移时的输入分布偏移。

这类项目解决的是感知链路里的基础误差。如果相机几何关系没有校准好，后面的视觉识别、抓取位姿估计、轨迹规划和策略学习都会被系统性偏差污染。

## 常见解决路线

| 方法 | 核心思想 | 适合处理的问题 |
| --- | --- | --- |
| System Identification | 用真实数据反推仿真参数 | 质量、摩擦、阻尼、刚度、接触模型误差 |
| Domain Randomization | 主动扩大仿真分布 | 视觉、材质、光照、物理参数和场景变化 |
| Domain Adaptation | 对齐源域和目标域特征分布 | 有仿真数据，也有少量或无标签真实数据 |
| Residual Policy | 在基础策略上学习补偿项 | 直接部署后出现系统性执行偏差 |
| Real2Sim2Real | 用真实系统表现反向优化仿真器 | 需要真实反馈参与仿真器或策略闭环优化 |

## 判断差距来源

定位 Sim2Real 失败时，可以先问四个问题：

- 失败是视觉识别错，还是动作执行错？
- 失败出现在接触瞬间、轨迹跟踪阶段，还是状态估计阶段？
- 仿真中是否包含真实系统的延迟、饱和、噪声和控制频率？
- 真实测试的初始状态分布是否超出仿真训练分布？

如果问题主要来自可解释物理参数，优先考虑系统辨识。如果问题主要来自场景和感知变化，优先考虑域随机化或域自适应。如果问题来自真实执行中的策略偏差，可以考虑残差策略或闭环 Real2Sim2Real。
