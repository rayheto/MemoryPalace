# 系统辨识

System Identification 是通过数学方法和实验数据构建真实系统动态模型的过程。本质是用可计算的参数化方程还原物理世界的运行规律，让仿真器更接近真实系统。

![[assets/system-identification-flow.svg|697]]

## 目标

系统辨识适合物理模型比较明确、参数可解释的场景，例如机械臂动力学、摩擦模型、接触模型、阻尼模型和执行器增益。

常见被辨识参数：

- 物体质量、质心、惯量。
- 关节摩擦、阻尼、刚度。
- 接触模型参数。
- 执行器延迟、增益和饱和。
- 视觉渲染中的材质反射和光照特性。

## 基本流程

1. 在真实系统上采集动作、状态和传感器数据。
2. 选择参数化模型，例如刚体动力学、接触模型或摩擦模型。
3. 在仿真中复现实验输入。
4. 最小化仿真轨迹和真实轨迹之间的误差。
5. 用识别出的参数更新仿真器。
6. 在更新后的仿真器中训练或微调策略。

可以把它理解为：不是让策略适应错误仿真器，而是先把仿真器调得更像真实系统。

## 两阶段验证

系统辨识常被组织成两阶段流程：

- 仿真训练阶段：在参数化模型中训练策略。
- 现实验证阶段：将策略部署到真实系统中验证。

如果真实验证失败，再根据真实数据更新参数，形成新的仿真器版本。

## 局限

- 接触、柔性物体、形变和复杂摩擦很难精确建模。
- 参数可能不可辨识，不同参数组合可能产生相似轨迹。
- 真实系统可能非平稳，例如磨损、温度漂移和夹具松动。
- 高保真辨识通常需要精心设计实验，数据成本不一定低。
